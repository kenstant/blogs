# 第1章 简单工厂模式

> **类别**：创建型（入门形态，常作为后续工厂方法、抽象工厂的前奏）  
> **痛点关键词**：`if-else` 或 `switch` 随运算种类膨胀、改一处牵一片  
> **一句话意图**：**把“new 谁”从业务里拿出去，用工厂集中创建。**

## 一、30 秒卡片（一眼回忆用）

- **它解决什么变化**：**运算 / 处理算法种类**在增加（加减乘除、以后还可能有别的）；**一次运算的输入结构**（两个数 + 一个符号）相对稳定。
- **关键角色**：
  - **抽象 `Operation`**：声明一次运算长什么样；  
  - **具体 `Add` / `Sub` / …**：各自实现算法；  
  - **`OperationFactory`（简单工厂）**：根据符号 **决定 new 哪个具体类**；  
  - **客户端 `Calculator`（或 `main`）**：只拿「抽象 + 多态结果」，不直接对每种运算 new。
- **它和谁容易混**：与后面要学的**策略**在代码结构上都像「一簇可替换子类 + 多态」——**区别在意图上**：这里突出**对象的创建/装配**；策略突出**可互换算法/行为**（下一章会再对读）。

> 一个月后只看本节就能回忆起这章 80% 内容。

## 二、不用模式时怎么写（先看烂代码）

**需求**：做一个控制台计算器，根据两个数和运算符，输出结果，支持 `+` `-` `*` `/`。

**v1 直觉写法**（书中）：

```java
package design_pattern.chap1.v1;

import java.util.Scanner;

public class CalculatorV1 {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        System.out.println("请输入数字a:");
        String a = sc.nextLine();
        System.out.println("请选择运算符号(+、-、*、/):");
        String b = sc.nextLine();
        System.out.println("请输入数字c:");
        String c = sc.nextLine();
        double d = 0d;
        if (b.equals("+")) {
            d = Double.parseDouble(a) + Double.parseDouble(c);
        }
        if (b.equals("-")) {
            d = Double.parseDouble(a) - Double.parseDouble(c);
        }
        if (b.equals("*")) {
            d = Double.parseDouble(a) * Double.parseDouble(c);
        }
        if (b.equals("/")) {
            d = Double.parseDouble(a) / Double.parseDouble(c);
        }
        System.out.println("结果是: " + d);
    }
}
```

**v1 的痛点**：

- 连续四个 `if`，**每次会多做几次无用比较**；命名也不清晰。  
- **除法未处理除数为零**。  
- 大量重复的 `Double.parseDouble()`。  
- 想加一种新运算，要在 **UI + 计算逻辑** 里同时改，**扩展点散**。

## 三、一步步演化

### 3.1 v2：改了什么

```java
package design_pattern.chap1.v2;

import java.util.Scanner;

public class CalculatorV2 {
    public static void main(String[] args) {
        try {
            Scanner sc = new Scanner(System.in);
            System.out.println("请输入数字a:");
            double numberA = Double.parseDouble(sc.nextLine());
            System.out.println("请选择运算符号(+、-、*、/):");
            String strOperator = sc.nextLine();
            System.out.println("请输入数字b:");
            double numberB = Double.parseDouble(sc.nextLine());
            double result = 0d;
            switch (strOperator) {
                case "+":
                    result = numberA + numberB;
                    break;
                case "-":
                    result = numberA - numberB;
                    break;
                case "*":
                    result = numberA * numberB;
                    break;
                case "/":
                    result = numberA / numberB;
                    break;
            }
            System.out.println("结果是: " + result);
        } catch (Exception e) {
            System.out.println("您的输入有错:" +  e);
        }
    }
}
```

**好在哪 / 还差什么**：

- **好**：`switch` 比四个 `if` 清晰，重复解析已收敛。  
- **差**：需求仍**全部塞在 `main`；加一种运算要改** `switch` / `main`，**可维护、可复用、可扩展、灵活**都还不够。书中也在这里引出「要写好程序，不止能跑」。

### 3.2 v3：再改什么

**把「怎么算」先抽到 `Operation.getResult`：**

```java
package design_pattern.chap1.v3;

public class Operation {
    public static double getResult(double numberA, double numberB, String operate) {
        double result = 0d;
        switch (operate) {
            case "+":
                result = numberA + numberB;
                break;
            case "-":
                result = numberA - numberB;
                break;
            case "*":
                result = numberA * numberB;
                break;
            case "/":
                result = numberA / numberB;
                break;
            // 其他操作...
        }
        return result;
    }
}
```

```java
package design_pattern.chap1.v3;

import java.util.Scanner;

public class CalculatorV3 {
    public static void main(String[] args) {
        try {
            Scanner sc = new Scanner(System.in);
            System.out.println("请输入数字a:");
            double numberA = Double.parseDouble(sc.nextLine());
            System.out.println("请选择运算符号(+、-、*、/):");
            String strOperator = sc.nextLine();
            System.out.println("请输入数字b:");
            double numberB = Double.parseDouble(sc.nextLine());
            double result = Operation.getResult(numberA, numberB, strOperator);
            System.out.println("结果是: " + result);
        } catch (Exception e) {
            System.out.println("您的输入有错:" +  e);
        }
    }
}
```

**好在哪 / 还差什么**：

- **好**：`main` 只关心参数与调用，**计算逻辑有单独归宿**。  
- **差**：若要在 `Operation` 里**新增一种算法（如开方）**，仍要**改**这个类里已有 `switch` 分支——**对修改开放的风险**仍在，容易在旧分支上改出 bug。书中用这一点引出下一版：把**每种算法拆成子类，新增算法尽量只加新类、少动旧代码。**

## 四、模式登场（终版）

**抽象运算 + 具体子类 + 简单工厂 + 客户端。**

**抽象与具体**（书中节选）：

```java
package design_pattern.chap1.v4;

public abstract class Operation {
    public double getResult(double numberA, double numberB) {
        return 0d;
    }
}
```

```java
public class Add extends Operation {
    @Override
    public double getResult(double numberA, double numberB) {
        return numberA + numberB;
    }
}
// Sub、Mul 同理
```

```java
public class Div extends Operation {
    @Override
    public double getResult(double numberA, double numberB) {
        if (numberB == 0) {
            System.out.println("除数不能为0");
            throw new ArithmeticException();
        }
        return numberA / numberB;
    }
}
```

**简单工厂**（书里的写法，`default` 处仍 `return null`，调用方若忘记判空会 NPE，属于后面「反例」里要补的坑）：

```java
package design_pattern.chap1.v4;

public class OperationFactory {
    public static Operation createOperate(String operate) {
        switch (operate) {
            case "+":
                return new Add();
            case "-":
                return new Sub();
            case "*":
                return new Mul();
            case "/":
                return new Div();
        }
        return null;
    }
}
```

**客户端**：

```java
package design_pattern.chap1.v4;

import java.util.Scanner;

public class CalculatorV4 {
    public static void main(String[] args) {
        try {
            Scanner sc = new Scanner(System.in);
            System.out.println("请输入数字a:");
            double numberA = Double.parseDouble(sc.nextLine());
            System.out.println("请选择运算符号(+、-、*、/):");
            String strOperator = sc.nextLine();
            System.out.println("请输入数字b:");
            double numberB = Double.parseDouble(sc.nextLine());
            Operation operation = OperationFactory.createOperate(strOperator);
            double result = operation.getResult(numberA, numberB);
            System.out.println("结果是: " + result);
        } catch (Exception e) {
            System.out.println("您的输入有错:" +  e);
        }
    }
}
```

**UML / 角色对应**：

| 模式里的说法 | 书中对应 |
| --- | --- |
| Product（抽象产品） | `Operation` |
| ConcreteProduct | `Add`、`Sub`、`Mul`、`Div` |
| Factory / Creator（简单） | `OperationFactory` |
| 使用端 | `CalculatorV4`（或独立 `Client` 包一层 UI） |

## 五、反例与代价

- **什么时候不该用**：具体算法**只有 1～2 种且基本不会再变**，为「好看」上抽象 + 多态 + 工厂，反而多几层跳转；**只有一处** `new`、创建规则不可能扩展时，直接写清楚即可。  
- **代价**：
  - 类/文件**数量变多**；读代码要**在类之间跳**；  
  - **多态与继承**是同事的上手成本；  
  - **书里的 `createOperate` 每加一种运算仍要改 `switch`**——**开放扩展的是具体算法类**这条线，**工厂**在经典 `switch` 实现里**往往是集中修改点**（与后面要学的**工厂方法**可对照）。  
  - `default: return null` 若调用方未判空，**运行期 NPE**；我后面在练习里改为 **`default` 抛 `IllegalArgumentException`**，把失败**钉在创建点**。
- **误用样子**：**上帝工厂**——一个 `create(String)` 里 `switch` 出彼此无关的几十种类（用户、订单、支付全塞一起），和「一簇可互相替换的同类算法」不是一回事，只是**把大 `if-else` 换了个房间**。

## 六、回到自己的代码

**与书中练习 A 的对应**：在烂代码 v1 上，用 **抽象 + 子类 + 工厂** 收拢「由符号决定 new 谁」，**main 只关心「抽象上的 `getResult`」**。

**练习 `FactoryPractice`（`default` 中抛出非法参数，避免悄悄 `null` 再 NPE）**：

```java
public class FactoryPractice {

    abstract static class Operation {
        abstract double getResult(double numberA, double numberB);
    }

    static class Add extends Operation {
        @Override
        public double getResult(double numberA, double numberB) {
            return numberA + numberB;
        }
    }

    static class Sub extends Operation {
        @Override
        public double getResult(double numberA, double numberB) {
            return numberA - numberB;
        }
    }

    static class OperationFactory {
        static Operation getOperation(String operator) {
            switch (operator) {
                case "+":
                    return new Add();
                case "-":
                    return new Sub();
                default:
                    throw new IllegalArgumentException("unknown op: " + operator);
            }
        }
    }

    public static void main(String[] args) {
        String op = args[0];
        double a = Double.parseDouble(args[1]);
        double b = Double.parseDouble(args[2]);
        Operation operation = OperationFactory.getOperation(op);
        double result = operation.getResult(a, b);
        System.out.println(result);
    }
}
```

**如果按本章思路再收紧**：`main` 用 `try/catch` 区分「非法符号」和「算式异常（如除零）」会更清晰，本书 v4 用大块 `catch (Exception` 是教学简化。
